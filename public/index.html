SDID JWKS and Client Assertion Service

Use this page to copy the public EC signing JWKS URL, the public RSA encryption JWKS URL, and curl commands for generating a signed client_assertion and requesting an access token.

Security reminder The JWKS endpoints are public because they expose public keys only. The /client-assertion endpoint signs JWTs using the private EC key, so it must remain protected with X-Assertion-Api-Key. Do not share your private key or assertion API key.

Public JWKS endpoints

EC · ES384 · sig

EC signing JWKS

Used by the SDID Authorization Server to verify the signed client_assertion JWT.

Open EC JWKS
RSA · RSA-OAEP-256 · enc

RSA encryption JWKS

Used only when another system needs your RSA public encryption key. Keep it separate from the EC signing JWKS.

Open RSA JWKS

Quick curl checks for JWKS

These commands help you confirm that both public JWKS endpoints are reachable after deployment. 

curl -s "{SITE_URL}/enrollment-creation-service/api/v1/requests/jwks" | jq

curl -s "{SITE_URL}/encryption-jwks" | jq

Create a client_assertion

The client_assertion is a signed JWT, not an encrypted value. The curl command below calls your protected Netlify endpoint and returns the JWT in the client_assertion field. 

curl -s -X POST "{SITE_URL}/client-assertion" \
  -H "Content-Type: application/json" \
  -H "X-Assertion-Api-Key: <ASSERTION_API_KEY>" \
  -d '{
    "clientId": "u5aw0txey983f0i97zh8hbhtqpahhael",
    "audience": "https://auth.sdid.uat:8443/auth/oauth2/token",
    "expiresInSeconds": 300
  }' | jq -r '.client_assertion'

Generate client_assertion and request access token

Use this complete command when you want to generate a fresh assertion and immediately exchange it for an OAuth access token. 

CLIENT_ASSERTION=$(curl -s -X POST "{SITE_URL}/client-assertion" \
  -H "Content-Type: application/json" \
  -H "X-Assertion-Api-Key: <ASSERTION_API_KEY>" \
  -d '{
    "clientId": "u5aw0txey983f0i97zh8hbhtqpahhael",
    "audience": "https://auth.sdid.uat:8443/auth/oauth2/token",
    "expiresInSeconds": 300
  }' | jq -r '.client_assertion')

curl -k -X POST "https://auth.sdid.uat:8443/auth/oauth2/token" \
  -H "Content-Type: application/x-www-form-urlencoded" \
  --data-urlencode "grant_type=client_credentials" \
  --data-urlencode "client_id=u5aw0txey983f0i97zh8hbhtqpahhael" \
  --data-urlencode "client_assertion_type=urn:ietf:params:oauth:client-assertion-type:jwt-bearer" \
  --data-urlencode "client_assertion=${CLIENT_ASSERTION}"

What to share with SDID

Item Value Purpose
Client ID u5aw0txey983f0i97zh8hbhtqpahhael OAuth client identifier.
EC JWKS URL Used to verify client_assertion.
Algorithm ES384 JWT signing algorithm for client authentication.
Authentication method PRIVATE_KEY_JWT OAuth client authentication method.
RSA JWKS URL Share only when RSA encryption public key is required.

Replace <ASSERTION_API_KEY> locally before running the curl commands. Never paste the assertion API key into this public page.
